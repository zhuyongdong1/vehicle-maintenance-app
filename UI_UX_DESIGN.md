# UI/UX 设计方案

## 1. 设计系统

### 配色
```
主色        #1E6BFF  蓝色，专业信赖
主色浅      #E8F0FE  卡片/选中背景
主色深      #1650CC  按压态
成功色      #10B981  收入、已完成
警告色      #F59E0B  提醒、待处理
危险色      #EF4444  支出、删除、逾期
中性色      白/灰阶梯  背景、文字、分割线
```

### 字体
- 中文：系统默认（PingFang SC / Roboto）
- 数字/金额：Tabular Lining（等宽数字，金额对齐）
- 层级：标题 20sp / 副标题 16sp / 正文 14sp / 辅助 12sp

### 圆角 & 阴影
- 卡片圆角 12px，按钮圆角 8px，输入框圆角 8px
- 卡片阴影：轻微浮起感 `0 2px 8px rgba(0,0,0,0.06)`
- FAB 阴影：`0 4px 16px rgba(30,107,255,0.3)`

### 图标
- Material Symbols Rounded（圆润风格）
- 统一 24px

## 2. 全局导航

### 底部导航栏（5 项）

| 图标 | 标签 | 路由 |
|------|------|------|
| 🏠 dashboard | 首页 | /dashboard |
| 🚗 directions_car | 车辆 | /vehicles |
| 🔧 build | 维修 | /records |
| 📒 book | 记账 | /ledger |
| ⚙️ settings | 设置 | /settings |

### 通用交互
- **FAB**：列表页右下角悬浮按钮，新增入口，主色+图标
- **下拉刷新**：所有列表页支持
- **空状态**：插画+引导文案+快捷入口按钮
- **长按**：列表项长按弹出删除确认
- **滑动返回**：详情页支持手势返回
- **SnackBar**：操作反馈（成功绿/失败红）
- **骨架屏**：加载中展示灰色占位块

## 4. 核心交互流程

### 流程 1：新车入场
```
扫码车牌 → OCR识别 → 自动填写车牌/车架号
  → 补充品牌/车主信息 → 拍照 → 保存
```

### 流程 2：维修完成
```
FAB新增维修 → 选车辆 → 选分类 → 填项目/费用
  → 设置下次保养提醒 → 勾选"记收入" → 保存
  → 自动创建一条收入记账
```

### 流程 3：日常记账
```
底部栏记账 → 选收入/支出 → 选分类 → 输金额
  → 关联维修记录（可选）→ 保存
```

### 流程 4：自定义分类
```
设置 → 维修类型 → 点+号 → 输入"空调维修" → 确认
  → 新建维修时下拉可选"空调维修"
```

## 5. 交互细节

| 场景 | 交互 |
|------|------|
| 删除操作 | 弹出确认对话框"确定删除XXX？此操作不可撤销" |
| 下拉刷新 | 所有列表页支持，刷新动画Loading |
| 搜索 | 输入即搜，300ms 防抖 |
| 金额输入 | 自动格式化为 ¥1,234.56，带小数点处理 |
| 日期选择 | Material DatePicker，中文日历 |
| 图片预览 | 点击放大，双指缩放 |
| 空状态 | 车辆/记录/记账为空时显示插图+引导按钮 |
| 金额颜色 | 收入绿色 `#10B981`，支出红色 `#EF4444` |
| 逾期提醒 | 红色标签+"超期N天"，临近黄色+"还剩N天" |
| OCR 扫描 | 弹出相机界面，拍照后显示识别结果+确认按钮 |

## 6. 动画

- **页面过渡**：Material 标准页面推入（300ms）
- **FAB**：列表滚动时缩小隐藏，停止时弹出
- **金额变化**：数字跳动动画（count-up）
- **删除**：列表项左滑露出删除按钮，或iOS风格滑动删除
- **扫描结果**：OCR 结果逐字弹出（typewriter 效果）
- **切换Tab**：内容区淡入淡出（200ms）
- **按钮点击**：Material Ripple 水波纹

## 7. 适配

- Android 主力：Material 3 原生体验
- Web 调试：最大宽度 480px 居中，两侧灰色背景，模拟手机
- 深色模式：后续可扩展（先做浅色）
- 安全区域：适配顶部状态栏 + 底部导航栏
